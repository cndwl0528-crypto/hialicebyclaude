COMPONENT FEATURES
================================================================================

1. ERROR BOUNDARY
   - Catches rendering errors and child errors
   - Shows friendly UI with "Try Again" button
   - Development mode shows error stack trace
   - HiAlice branding and colors
   - No production error exposure

2. LOADING SKELETONS
   - SkeletonCard: Image + text placeholder
   - SkeletonText: Configurable text lines
   - SkeletonAvatar: 4 sizes (sm/md/lg/xl)
   - SkeletonChat: Two variants (alice/student)
   - PageLoader: Full-page spinner with branding
   - SkeletonGrid: Auto-grid of skeletons
   - Pure CSS shimmer animation (performant)

3. OFFLINE BANNER
   - Automatic online/offline detection
   - Yellow banner with 📡 emoji
   - Auto-hides when reconnected
   - Child-friendly message
   - No hydration issues

4. OFFLINE SUPPORT
   - Service Worker with 3 caching strategies:
     * Network-first for API calls
     * Cache-first for static assets
     * Fallback HTML for offline pages
   - Offline page with status indicator
   - Auto-redirect on reconnect
   - What you can do offline guidance

5. PWA FEATURES
   - Installable on home screen (iOS/Android)
   - Custom splash screens (Android)
   - App shortcuts (Start Reading, Vocabulary)
   - Maskable icons for adaptive displays
   - Theme colors (#4A90D9)
   - Background color (#F5F7FA)

6. SECURITY
   - Service Worker cache headers
   - X-Content-Type-Options: nosniff
   - X-Frame-Options: DENY
   - X-XSS-Protection: 1; mode=block
   - Environment variable isolation
   - No secrets in version control

7. DOCKER & DEPLOYMENT
   - Multi-stage build (frontend + backend)
   - Node 20 Alpine (lightweight)
   - dumb-init for signal handling
   - Health checks for both services
   - Automatic restart on failure
   - PostgreSQL included for local dev

SUPPORT & TROUBLESHOOTING
================================================================================

SERVICE WORKER NOT REGISTERING?
1. Verify HTTPS (required for production)
2. Check /sw.js is accessible
3. Look for errors in browser console
4. Try hard refresh (Ctrl+Shift+R)

OFFLINE MODE NOT WORKING?
1. Verify SW is installed (DevTools → Application)
2. Load a page (SW caches it)
3. Go offline (DevTools → Network → Offline)
4. Reload page

DOCKER BUILD FAILS?
1. docker system prune --all
2. docker-compose up -d --build
3. Check error logs

DEPLOYMENT FAILS?
1. Verify .env.local exists and has all vars
2. Check port 3001 is available
3. Review logs: docker logs hialice-app
4. Run: bash scripts/deploy.sh production
